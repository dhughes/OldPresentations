Test Case Assertions

This is a listing of all the assertions available to any class that extends org.cfcunit.framework.TestCase. Each method has a general description and a listing of its arguments details (name,type,required,default).

Identity Assertions
assertSameComponent
Asserts that two components refer to the same object. If they are not an AssertionFailedError is thrown with the given message. If the two variables do not refer to the same object, it helps if both objects have a method called 'toStringValue()' that can be used to create a textual representation of the objects in the failure message.
expectedWEB-INF.cftags.componentrequired 
actualWEB-INF.cftags.componentrequired 
failureMessagestringoptionalempty string
